# Novel Recall 录屏脚本

推荐录屏时长：45 到 90 秒。

## 录屏前准备

1. 执行构建：

```powershell
cd D:\code\NovelRecall
npm run build
```

2. 在 `chrome://extensions` 加载 `D:\code\NovelRecall\dist`。
3. 准备好 `examples/demo-novels.json`。
4. 准备 SiliconFlow API Key，但不要在录屏中暴露完整 Key。
5. 如果需要展示从零流程，可先在 options 页面清空本地小说库。

## 导入 demo 数据

1. 打开 Novel Recall options 页面。
2. 展示标题 `Novel Recall 设置`。
3. 点击导入 JSON。
4. 选择 `examples/demo-novels.json`。
5. 展示导入后小说数量和列表。

推荐讲解：

```text
这里导入的是项目内置的手写虚构 demo 数据，不是真实平台数据，也不包含小说正文。
```

## 配置 SiliconFlow API Key

1. 在设置区输入 API Key。
2. 点击保存。
3. 展示“已保存”状态。
4. 如需安全录屏，可遮挡输入框或使用后期打码。

推荐讲解：

```text
API Key 只保存在 chrome.storage.local，不写入小说数据，也不会导出。
```

## 批量 AI 分析

1. 点击“批量分析未分析小说”。
2. 展示 loading 状态。
3. 展示生成后的摘要、剧情关键词、人设标签、题材标签。

推荐讲解：

```text
AI 分析只基于导入的标题、简介和标签，生成结构化信息，不采集任何平台页面。
```

## 批量生成向量

1. 点击“批量生成未生成向量”。
2. 展示向量生成状态。
3. 展示“已生成向量”，不要展示完整 embedding 数组。

推荐讲解：

```text
小说向量保存在 IndexedDB，用于后续本地相似度排序。
```

## popup 剧情检索演示

1. 打开 popup。
2. 确认顶部有 Novel Recall、小说数量和设置按钮。
3. 选择“剧情检索”。
4. 输入：

```text
女主重生回高中，男主是她同桌，前世一直暗恋她
```

5. 点击“剧情检索”。
6. 展示 similarity 分数和排序结果。

推荐讲解：

```text
这里不是关键词硬匹配，而是把查询和本地小说向量做 cosine similarity 排序。
```

## popup 关键词检索演示

1. 切换到“关键词检索”。
2. 输入：

```text
机甲
```

3. 展示本地字段匹配结果。

推荐讲解：

```text
关键词检索完全本地完成，不调用 SiliconFlow。
```

## AI 猜书演示

1. 切换到“AI 猜书”。
2. 输入较完整剧情：

```text
女主重生回高中，男主是她同桌，前世一直暗恋她，这一世两人一起准备高考，偏校园甜文
```

3. 点击“AI 猜书”。
4. 展示候选卡片、匹配元素、置信度和“AI 推测，未验证”。
5. 再输入过短或无效内容，例如：

```text
不知道什么小说
```

6. 展示追问提示，不展示占位候选。

推荐讲解：

```text
AI 猜书不是数据库检索，也不联网搜索。信息不足时会追问，不会硬编未知小说。
```

## 导出 JSON 演示

1. 回到 options 页面。
2. 点击导出 JSON。
3. 简短说明导出包含小说元数据、AI 结果和 embedding。
4. 强调导出不包含 API Key。

推荐讲解：

```text
导出文件用于本地备份，不包含 API Key，也不包含完整小说正文。
```

## 结尾建议

最后用 5 秒总结：

```text
Novel Recall 把用户自己的小说记录变成可搜索的本地知识库，核心是本地存储、AI 结构化、向量检索和清晰的数据边界。
```
